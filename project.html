Hardware Rundown

This system designed around the Raspberry Pi Pico microcontroller uses a combination of a sound sensor and an LCD display to monitor the noise levels within an environment, providing a visual alert when the detected sound exceeds a preset threshold. Written in MicroPython, this setup leverages the simplicity and flexibility of Python in a microcontroller environment.

There were 3 main hardware components involved in making this project:

  • Raspberry Pi Pico and Shield
  • Sound Sensor
  • LCD Display

Software Rundown

The MicroPython code can be broken down into several main blocks:

  • Library Imports
    • -lcd1602: A custom library downloaded off the moodle page and uploaded directly to the Pico designated for controlling the LCD1602 display via I2C.
    -machine: Provides access to various hardware components like I2C, ADC (Analog to Digital Converter), and Pin objects.
    -utime: For implementing delays in the program.
  • Initial Setip
    • -The sound sensor's analog output is connected to pin 26 of the Pi Pico, and is set up to provide a digital representation of the sound (analog) using the ADC (Analog to Digital Converter)
    -The I2C communication setup for the LCD involves specifying the Pi Pico's I2C1 interface and the SCL and SDA pins (7 and 6, respectively), with a communication frequency of 400 kHz.
  • Main Loop
    • -Sound Detection: The loop continuously monitors the analog value read from the sound sensor. By calling adc.read_u16(), it reads a 16-bit value that represents the intensity of the sound detected.
    Threshold Check: It then compares this value to a predefined threshold (25000 in this case). This threshold represents the sensitivity of the sound detection. The higher the threshold, the louder the sound has to be to trigger the detection. Adjusting this threshold is key to calibrating the system according to the noise levels one intends to monitor.
    Display Message: If the sound level exceeds the threshold, the LCD first clears any previous message, prints "High! " to indicate a high sound level, waits for 1 second (to allow the user to read the message), and then clears the display again. This immediate feedback loop acts as an alert mechanism.
    Delay: There's a brief delay (0.1 seconds) at the end of the loop to prevent excessive updating, which can make the display difficult to read and also unnecessarily burden the Pi Pico's processor whilst also making sure it is updating often enough to accuratly measure the sound level.
